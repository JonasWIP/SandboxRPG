Full World Design — Terrain & Chunk System

Updated with voxel terrain, generated structures, and decorations. Review each section.

World Structure

🏰 Spawn Town
  • ±5 chunks from origin (320×320m)
  • Pre-generated on Init()
  • No building, no terrain edit, no PvP
  • NPCs: shopkeepers, quest-givers, innkeeper
  • Generated structures: market, blacksmith, inn, notice board
🌍 Outer World
  • Lazy-generated as players explore
  • 5 blended biomes (noise map)
  • Full voxel terrain — dig & fill anywhere
  • Resource nodes on surface
  • Ore veins underground
  • Random POIs & ruins
🌀 Alt Worlds
  • Accessed via portals
  • Separate SpacetimeDB instance
  • Different generation algorithm
  • Out of scope for this spec

Voxel Terrain (Astroneer-style)

Chunks are 32×32×32 voxel grids. Each voxel is a density value (signed byte, –127 to +127). Positive = solid, negative = air. The isosurface at 0 is the terrain surface — rendered with marching cubes on the client.

Underground chunks extend to ChunkY = –4 (128m deep). Surface is ChunkY = 0. Sky chunks (ChunkY > 0) are mostly air.

Ore veins are a second channel stored alongside density — an ore type byte per voxel (0 = none). When a player digs a voxel containing ore, the ore drops as an item.

// 32KB per chunk
WorldChunk {
  ChunkKey (PK),
  ChunkX, ChunkY, ChunkZ,
  BiomeType,
  VoxelDensity byte[32768],
  VoxelOre byte[32768],
  IsTownZone,
  IsGenerated
}

Server-Side Entities

// Trees, rock outcrops, herb bushes
ResourceNode {
  Id, ChunkKey,
  NodeType, ResourceType,
  PosX, PosY, PosZ,
  Health, MaxHealth,
  RespawnAt
}
// Wells, crates, mushroom patches
WorldObject {
  Id, ChunkKey,
  ObjectType,
  PosX, PosY, PosZ,
  State, RespawnAt
}
// Ruins, bandit camps, shrines
GeneratedStructure {
  Id, StructureType,
  AnchorChunkKey,
  PosX, PosY, PosZ, RotY,
  IsCleared
}

Decorations (rocks, flowers, fallen logs) are client-side only — generated deterministically from chunk seed + biome. No server storage needed.

Generation Flow

  1. Init(): Store WorldConfig (seed). Pre-generate all town-zone chunks. Place town structures (inn, market, blacksmith). Spawn town NPCs.
  2. Player moves → server computes chunk column at player XZ position.
  3. Missing chunk? → Generate: sample biome noise → build voxel density + ore channels → insert surface ResourceNodes and WorldObjects → maybe place GeneratedStructure.
  4. Client subscribes to WorldChunk → runs marching cubes → renders terrain mesh. Spawns node visuals from ResourceNode / WorldObject / GeneratedStructure tables. Draws client-side decorations from seed.
  5. Player digs/fills → HarvestTerrain reducer updates VoxelDensity in WorldChunk → clients rebuild mesh.
  6. Scheduled reducer → restores ResourceNode health and WorldObject state when RespawnAt is reached.
